# SDKs

Every published package in the OrangeCheck ecosystem, with what it does, which
protocol it serves, and a link to the full API reference. All packages are
MIT-licensed and live in the
[`oc-packages`](https://github.com/orangecheck/oc-packages) monorepo (TypeScript)
or [`oc-py`](https://github.com/orangecheck/oc-py) (Python).

> **The detailed per-export, per-type, per-signature reference under each
> TypeScript package is auto-generated from the source via
> [TypeDoc](https://typedoc.org/) on every release.** TSDoc comments in the
> `.ts` source are the source of truth; per-package READMEs are short
> taglines + install + a 30-second example. Hand-written narrative
> (quickstart, "why this protocol") lives under each protocol's section
> ([OC Attest](/attest), [OC Pledge](/pledge), etc.).

## Semver

Everything below 1.0 is pre-stable. Minor bumps CAN change exported types;
patch bumps won't. Check the per-package README before upgrading; diff the
changelog if something looks ambiguous.

`@orangecheck/sdk`, `@orangecheck/agent-core`, `@orangecheck/pledge-core`,
`@orangecheck/lock-core`, `@orangecheck/stamp-core`, `@orangecheck/vote-core`
are at v1.0.0+ — wire-format-stable; type changes are semver-respecting.

## Cross-implementation conformance

The TypeScript SDK and the Python SDK ship vendored copies of the same
canonical test vectors from the protocol repos. CI re-runs both suites on
every push and fails on drift. This means: a canonical message built in TS
and a canonical message built in Python for the same inputs are
**byte-identical**. Ditto attestation IDs, poll IDs, stamp envelope IDs,
pledge IDs. If you ever observe otherwise, it's a bug — please file it.

## Local generation of the auto-gen reference

Clone [`oc-packages`](https://github.com/orangecheck/oc-packages) and
[`oc-docs`](https://github.com/orangecheck/oc-docs) as siblings, then:

```bash
cd oc-packages
yarn install
yarn docs:gen <pkg>     # regenerate one package
yarn docs:gen:all       # regenerate every package
yarn docs:check         # exit 1 if regenerated docs differ from committed oc-docs
```

The generator writes into `../oc-docs/src/pages/sdk/<pkg>/`. CI gates
drift via Phase 3.5: release tags auto-PR the regenerated tree to oc-docs
(gated on `OC_DOCS_BOT_TOKEN`).
